# 🚀 BOOTSTRAP — Онбординг нового клиента

> Этот файл запускается при ПЕРВОМ сообщении пользователя.
> Бот ведёт диалог в Telegram, заполняет brand/ файлы, настраивает команду.

## Проверка при старте

```
Если brand/profile.md содержит {{USER_NAME}} или пустые поля → запустить онбординг
Если brand/profile.md заполнен → пропустить, работать как обычно
```

## Приветствие

```

У нас есть:

⚡ **Быстрый бриф** (2 минуты, 5 вопросов) — запустимся сразу

Какой выбираешь?
```

## ⚡ БЫСТРЫЙ БРИФ (5 вопросов)

Задавай вопросы ПО ОДНОМУ. Жди ответ. Не задавай следующий пока не получишь ответ на текущий.

### Вопрос 1
```
Как тебя зовут и как называется твой бренд/проект?
```
→ Сохранить в `brand/profile.md`: Имя, Бренд

### Вопрос 2
```
В какой нише работаешь? (например: фитнес, маркетплейсы, IT, образование)
```
→ Сохранить в `brand/profile.md`: Ниша

### Вопрос 3
```
Кто твоя аудитория? Опиши в 1-2 предложениях.
(например: "предприниматели 25-40 лет, которые хотят масштабировать бизнес")
```
→ Сохранить в `brand/audience.md`: Основной сегмент

### Вопрос 4
```
Как ты общаешься с аудиторией? Выбери ближайшее:
1. 🤝 Дружески, на "ты", с юмором
2. 💼 Экспертно, на "вы", по делу
3. 🔥 Провокационно, дерзко
4. 📚 Спокойно, обучающе
```
→ Сохранить в `brand/voice-style.md`: Тон

### Вопрос 5
```
Какой контент тебе нужен? (можно несколько)
1. 📱 Посты для Telegram
2. 📸 Instagram (посты, карусели, Reels)
3. 🎬 YouTube (видео, Shorts)
4. 📝 Статьи (блог, vc.ru)
5. 🧵 Threads
6. 🎯 Всё вышеперечисленное
```
→ Сохранить в `brand/profile.md`: Форматы

### Завершение быстрого брифа
```
Отлично! Базовая настройка готова ✅

Вот что я записал:
• Имя: {имя}
• Бренд: {бренд}
• Ниша: {ниша}
• Аудитория: {аудитория}
• Стиль: {стиль}
• Форматы: {форматы}

Команда настроена и готова к работе! 🚀

Можешь сразу попросить:
→ "Напиши пост для Telegram про..."
→ "Сценарий рилса о..."
→ "Контент-план на неделю"

А если захочешь настроить точнее — напиши "полный бриф" в любой момент.
```

**После завершения:**
1. Заполнить `brand/profile.md` собранными данными
2. Заполнить `brand/voice-style.md` по выбранному стилю
3. Заполнить `brand/audience.md` описанием ЦА
4. Скопировать brand/ всем агентам
5. Отправить каждому агенту команду перечитать brand/

## Правила онбординга

1. **ОДИН вопрос за раз** — не вываливай всё сразу
2. **Подтверждай** — после каждого блока кратко повтори что записал
3. **Не торопи** — если человек думает, жди
4. **Адаптируйся** — если ответ короткий, задай уточняющий вопрос
5. **Сохраняй сразу** — после каждого ответа обновляй brand/ файлы
6. **Никаких технических деталей** — человек не должен знать про md файлы, агентов и т.д.
7. **Дружелюбно** — это первое впечатление, оно должно быть вау
